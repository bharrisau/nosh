## Cross-Phase Integration (4/4 WIRED — opus integration checker)

1. **Identity is the spine** — one `nosh_auth::NoshPublicKey` (eq by `key32:[u8;32]`) flows from the single TLS-handshake extraction (`server.rs:215/890`) into P5 persistence/cap key (`registry.rs:593`) and P6 reattach second factor (`registry.rs:518/541`). WIRED.
2. **Persistence → reattach** — P6 reattach resumes the exact P5 `SessionSlot` + `SequencedOutputBuffer`; token is an identity-scoped selector with opaque no-oracle errors; PTY writer hand-back awaited before orphan. WIRED.
3. **Migration vs reattach disjoint** — P7 migration stays in the QUIC layer (same `conn`, never re-enters `handle_connection`); P6 reattach is a new connection. Same single registry, cannot collide. WIRED.
4. **Windows client → server E2E** — FileSigner (P8) and AgentSigner share `RawEd25519Signer` → identical SPKI → server `NoshPublicKey`; client speaks unchanged nosh-proto; `#[cfg]` splits are signer/resize-source only, not protocol forks. WIRED + human-validated.
