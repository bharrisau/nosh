# Plan 02-02: SSH-key SPKI pinning verifiers

## Objective
Replace `PlaceholderServerVerifier::verify_server_cert` with SSH-key SPKI
pinning against known_hosts (TOFU on first contact, hard-fail on mismatch), and
add a server-side `AuthorizedKeysVerifier` (`ClientCertVerifier`) pinning the
client cert SPKI against authorized_keys. Both keep REAL signature verification
delegated to the CryptoProvider (never stubbed). Covers AUTH-01/02/03.

## must_haves
- truths:
  - "Unknown client key → verify_client_cert returns Err (AUTH-01, D-03)"
  - "known_hosts mismatch → verify_server_cert returns Err; unknown host → TOFU write + Ok (AUTH-02, D-01/D-02)"
  - "verify_tls13_signature is REAL CryptoProvider delegation, never a stub (AUTH-03, PITFALL 5)"
  - "supported_verify_schemes returns only ED25519 (D-12)"

## Tasks

<task id="1" type="execute">
<action>
Rename/repurpose verifier.rs. Replace `PlaceholderServerVerifier` with
`HostKeyVerifier` implementing `rustls::client::danger::ServerCertVerifier`. It holds
the known_hosts path, the connect host string, and `Arc<CryptoProvider>`. `verify_server_cert`
extracts the peer SPKI (`keys::extract_spki_from_cert`), looks up the host in known_hosts:
on a recorded entry, compare SPKI and return `Err(Error::General("host key mismatch ..."))`
on mismatch; on no entry, append the key (TOFU) and return Ok; on match return Ok. Keep
`verify_tls12_signature`/`verify_tls13_signature` delegating to `verify_tls13_signature`/`verify_tls12_signature`
helpers exactly as the placeholder did. `supported_verify_schemes` returns `vec![SignatureScheme::ED25519]`.
Provide a constructor `HostKeyVerifier::new(known_hosts_path, host, provider)`.
</action>

<task id="2" type="execute">
<action>
Add `AuthorizedKeysVerifier` implementing `rustls::server::danger::ClientCertVerifier`.
It holds `Vec<NoshPublicKey>` (pinned SPKIs) and `Arc<CryptoProvider>`.
`offer_client_auth` → true, `client_auth_mandatory` → true, `root_hint_subjects` → `&[]`.
`verify_client_cert` extracts the peer SPKI and returns
`Ok(ClientCertVerified::assertion())` iff it equals one pinned key's SPKI, else
`Err(Error::InvalidCertificate(CertificateError::ApplicationVerificationFailure))`.
`verify_tls12_signature`/`verify_tls13_signature` delegate to the CryptoProvider (REAL).
`supported_verify_schemes` → `[ED25519]`. Constructor `AuthorizedKeysVerifier::new(keys, provider)`.
</action>
