# Plan 03-02: Server-side PTY session pump

Replace the echo loops in `handle_connection` with a real PTY login-shell session. Introduce a
discrete `session` module (M3 reattach seam). All session traffic is `Message` frames over a
single bidi stream (D-01). Deny-by-default env (FOOTGUN-1/2). SIGHUP+reap on disconnect
(no zombies). Exit code → `SessionClose` → clean QUIC close (SESS-08/09).

<action>
Create `crates/nosh-server/src/session.rs` and declare `pub mod session;` in `lib.rs`.

Define:
- `pub struct PasswdEntry { pub name: String, pub shell: String, pub home: String }` and
  `pub fn lookup_self() -> PasswdEntry` that uses `nix::unistd::{geteuid, User::from_uid}` to
  read the current euid's passwd entry; fall back to env (`$USER`, `$SHELL` or `/bin/sh`,
  `$HOME` or `/`) when unavailable. Accept an optional `shell_override: Option<&str>` so the
  server `--shell` flag / tests can force `/bin/sh`.
- `pub const ENV_WHITELIST_EXACT: &[&str] = &["TERM","LANG","TZ"];` and treat any key starting
  with `LC_` as allowed. EXPLICITLY documented deny list (never set): keys starting with `LD_`,
  `DYLD_`, and the exact keys `BASH_ENV`,`ENV`,`IFS`,`SHELLOPTS`,`PYTHONPATH`,`NODE_OPTIONS`,
  `SSH_AUTH_SOCK`. Implement as deny-by-default: build env from scratch, only copy a client var
  if it is in the allow set (exact or `LC_` prefix) — everything else is dropped, so the deny
  list is the documentation of what an attacker would try, not a filter that could be bypassed.
- `pub fn build_child_env(passwd: &PasswdEntry, client_env: &[(String,String)]) -> Vec<(String,String)>`
  returning the baseline (`HOME`,`USER`,`LOGNAME`,`SHELL`,`PATH`) from passwd + server PATH
  (default `/usr/local/bin:/usr/bin:/bin` if `$PATH` empty), then the whitelisted client vars.
- `pub struct Session { pub session_id: Uuid, pub identity: Option<NoshPublicKey>, master:
  Box<dyn portable_pty::MasterPty + Send>, child: Box<dyn portable_pty::Child + Send + Sync>,
  child_pid: Option<u32>, pub idle_since: Option<std::time::Instant> }`. (Import
  `nosh_auth::NoshPublicKey` for identity; it may be `None` for this spike since Phase 2 does
  not yet surface the peer key into the connection — pass `None` and note the limitation.)
- `pub fn open(client: &SessionOpen-equivalent fields, shell_override: Option<&str>) ->
  anyhow::Result<(Session, Box<dyn Read+Send>, Box<dyn Write+Send>)>`: builds `PtySize` from
  cols/rows, `openpty`, builds `CommandBuilder::new(shell)`, sets `arg0(format!("-{}", basename))`
  for a login shell, `cwd(home)`, `env_clear()`, then applies `build_child_env`, spawns,
  captures `child.process_id()`, returns the session plus the cloned reader and taken writer.
- `impl Session`: `pub fn resize(&self, cols: u16, rows: u16) -> anyhow::Result<()>` calling
  `self.master.resize(PtySize{..})`; `pub async fn wait_exit(&mut self) -> i32` that moves/clones
  the child wait onto `tokio::task::spawn_blocking` and returns the exit code (use
  `child.wait()` → `ExitStatus`; portable-pty `ExitStatus::exit_code()` returns u32 → cast to i32).
- `pub async fn shutdown(child_pid: Option<u32>, child: Box<dyn Child+Send+Sync>)`: send SIGHUP
  via `nix::sys::signal::kill(Pid::from_raw(pid as i32), Signal::SIGHUP)` (ignore errors), then
  `spawn_blocking(move || child.wait())` to reap — guaranteeing no zombie (PITFALL). Provide
  this so the connection handler can call it on disconnect.
Because `portable-pty::Child::wait` takes `&mut self`, structure `Session` so the child can be
owned by the wait/shutdown path (e.g. store `Option<Box<dyn Child..>>` and `take()` it). Keep it
simple and correct over clever.
</action>

<task>
<objective>Drive the session in handle_connection; remove the echo loops; clean close + exit code.</objective>
<read_first>

<action>
In `server.rs`:
- Delete `stream_echo_loop` and `datagram_echo_loop` (and their TRANS-0x doc comments).
- Add a `shell_override: Option<String>` parameter threaded from `run_accept_loop` →
  `handle_connection` (default `None`; `main` wires the `--shell` flag). Update `run_accept_loop`
  signature and its single call site, and `main.rs`.
- In `handle_connection`, after auth completes and the permit is dropped: `conn.accept_bi()`
  to get the session's `(SendStream, RecvStream)`. Read the first frame with
  `nosh_proto::read_message(&mut recv)`; require it to be `Message::SessionOpen { term, cols,
  rows, env }` (else close with an app error code + reason "expected SessionOpen").
- Look up the shell (`session::lookup_self(shell_override)`), `session::open(..)` to spawn the
  PTY shell; capture `session_id` (Uuid::new_v4), `username`, `child_pid`.
- Enter a `tracing::info_span!("session", %session_id, %peer, username = %username)`. Log
  "session open" with term/cols/rows.
- Spawn the OUTPUT pump: a `spawn_blocking` thread reads up to 8 KiB from the PTY reader in a
  loop and forwards chunks over an `mpsc::channel` (or `tokio::sync::mpsc` via
  `blocking_send`); an async task drains the channel and `write_message(&mut send,
  &Message::PtyData{data})`. On reader EOF, stop.
- Run the INPUT pump: async loop `read_message(&mut recv)`; on `PtyData{data}` write bytes to
  the PTY writer (writer is blocking — wrap writes in `spawn_blocking` or hold an
  `Arc<Mutex<Box<dyn Write+Send>>>` and call `write_all`+`flush`); on `Resize{cols,rows}` call
  `session.resize(cols,rows)` and log; on `SessionClose` or stream-closed, end.
- Run a wait task: `session.wait_exit().await` yields the shell exit code.
- Use `tokio::select!` over {wait_exit, input pump error/close, output pump end}. When the
  shell exits (wait_exit wins): send `Message::SessionClose { exit_code, reason: "shell exited" }`
  via `write_message`, `send.finish()`, then `conn.close(0u32.into(), b"shell exited")`.
- On client disconnect first (input pump sees stream/conn closed): call
  `session::shutdown(child_pid, child)` to SIGHUP+reap (no zombie), then return Ok.
- Reuse/keep `clean_exit` for treating orderly teardown as Ok.
Keep error handling with `anyhow` and `tracing::warn!` consistent with the existing style.
</action>
