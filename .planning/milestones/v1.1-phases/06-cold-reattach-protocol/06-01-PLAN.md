<objective>
Add the five additive `Message` variants for cold reattach (D-03) to `nosh-proto`, encoded compatibly with the existing length-delimited postcard codec, and prove they round-trip. This is the wire-protocol foundation every other Phase 6 plan builds on. No behavior change to existing variants.
</objective>

<context>
postcard encodes enum variants by their declaration-order discriminant (varint). Appending new variants to the END of `Message` is backward-compatible; inserting or reordering is NOT. The existing variants are, in order: `SessionOpen`, `PtyData`, `Resize`, `SessionClose`. The five new variants MUST be appended after `SessionClose`.

The token is the reattach selector (D-05): a fixed `[u8; 16]` sourced later from `Uuid::new_v4().into_bytes()`. Keep it `[u8;16]` (compact wire, `Copy`) — NOT a `String`. `ReattachErr` is deliberately fieldless (D-07): a wrong-identity rejection and a bad-token rejection must produce byte-identical frames so there is no session-existence oracle.

`ReattachOk` carries `replaying_from_seq` (the seq the server will replay from — `last_acked_seq+1` normally, or `lowest_retained_seq` on truncation) and `truncated: bool` (D-09). `Ack` carries the highest applied seq (D-08).
</context>

<task name="Add five reattach Message variants">
<read_first>
- crates/nosh-proto/src/messages.rs (current Message enum — append after SessionClose)
- crates/nosh-proto/src/codec.rs (postcard encode/decode + existing round-trip tests to mirror)
</read_first>
<action>
In `crates/nosh-proto/src/messages.rs`, append these five variants to the `Message` enum AFTER the existing `SessionClose` variant (declaration order is the postcard discriminant — do not reorder existing variants):

- `SessionOpened { token: [u8; 16] }` — server→client immediately after a successful fresh SessionOpen; delivers the initial reattach token (D-03).
- `Reattach { token: [u8; 16], last_acked_seq: u64 }` — client→server, the FIRST frame on a reconnect in place of SessionOpen (D-03). Document the `last_acked_seq` convention in a doc-comment: it is the highest output sequence number the client has APPLIED; the server replays `seq > last_acked_seq`. (The next plan that implements replay must honor exactly this convention.)
- `ReattachOk { new_token: [u8; 16], replaying_from_seq: u64, truncated: bool }` — server→client on success; `new_token` is the rotated single-use token (D-05); `replaying_from_seq` is the first seq the server will replay; `truncated` is true when the requested resume point fell below the buffer's lowest retained seq (D-09).
- `ReattachErr` — server→client on ANY reattach failure. FIELDLESS and uniform/opaque (D-07): unknown token, expired token, wrong identity, or already-attached session all return this identical variant. Add a doc-comment stating it must never carry a reason/code (no oracle).
- `Ack { seq: u64 }` — client→server, periodic; highest sequence number the client has received+applied (D-08).

Keep the existing derive list (`Debug, Clone, PartialEq, Eq, Serialize, Deserialize`) on the enum. The token bytes will be visible in the enum's derived `Debug`; add a doc-comment on `SessionOpened`/`Reattach`/`ReattachOk` warning that the token MUST NOT be logged (callers log only the identity fingerprint per D-07) — enforcement lives in the server plan, but the warning belongs at the type.
</action>
<acceptance_criteria>
- `messages.rs` contains `SessionOpened`, `Reattach`, `ReattachOk`, `ReattachErr`, `Ack` declared after `SessionClose`
- `ReattachErr` has no fields (it is `ReattachErr,` not `ReattachErr { .. }`)
- `Reattach` has fields `token: [u8; 16]` and `last_acked_seq: u64`
- `ReattachOk` has fields `new_token: [u8; 16]`, `replaying_from_seq: u64`, `truncated: bool`
- `cargo build -p nosh-proto` succeeds
</acceptance_criteria>
</task>

<action>
In `crates/nosh-proto/src/codec.rs` `#[cfg(test)] mod tests`, add a `#[tokio::test]` (e.g. `reattach_variants_round_trip`) that constructs each of the five new variants with non-trivial values (a token like `[0xAB; 16]`, a `last_acked_seq`/`seq` like `12345`, `replaying_from_seq` like `42`, `truncated: true`), writes each via `write_message` into a `Vec<u8>`, reads it back via `read_message` from a `std::io::Cursor`, and asserts equality. Add a focused assertion that `Message::ReattachErr` encodes to a frame whose body is byte-identical regardless of context (encode it twice and assert the two frames are equal) — this is the no-oracle property at the wire level. Also assert that decoding a `SessionClose` frame (an existing variant) still yields `SessionClose` after the enum was extended (proves appending did not shift existing discriminants) by encoding a `SessionClose` and decoding it back.
</action>
