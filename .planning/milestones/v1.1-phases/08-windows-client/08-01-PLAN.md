<objective>
Add `FileSigner` to `nosh-auth` — a third `RawEd25519Signer` impl (sibling of `AgentSigner`
and `InProcessEd25519Signer`) that loads an on-disk OpenSSH Ed25519 private key, detects and
rejects passphrase-encrypted keys with clear guidance, holds key material in the narrowest
possible scope with the transient seed zeroized, and signs the TLS `CertificateVerify`
in-process. Add `ClientIdentity::from_identity_file` as a third (platform-agnostic, opt-in)
constructor alongside `from_signer`/`from_agent`. Unblock the Windows build by gating the
Unix-only `ssh-agent-client-rs` dependency (and `AgentSigner`) behind `cfg(unix)` — leaving
all the SPKI/cert/signing logic identical on every platform. Prove it end-to-end with a
Linux headless test that writes a key to disk and authenticates through it.

Purpose: This is the auth foundation the Windows client stands on (WIN-02) and the change
that lets the crate cross-compile to Windows (WIN-01). Everything in Plan 02 (the main.rs
auth-path selection, the crossterm bump, the cross-compile gate) depends on `from_identity_file`
and the ssh-agent gating existing first. The `--identity-file` flag working on Linux (D-03)
lets this whole signer be validated in headless CI without a Windows box.
</objective>
