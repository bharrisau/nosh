<objective>
Three robustness/hygiene fixes found in live validation: (1) make `authorized_keys` parsing tolerant of non-Ed25519 / malformed lines (sshd behavior) so a real-world file with mixed keys still authorizes the supported ones; (2) give the client a connect timeout so it fails fast with a clear message when no server answers; (3) silence the Windows-only unused `PathBuf` import warning in the signer.

Purpose: Real `authorized_keys` files routinely contain RSA/ECDSA entries; today the first such line rejects the whole file. A missing connect timeout hangs the client. The import warning is a clean-build hygiene gap from Phase 8.
Output: A warn+skip parser with a unit test, a timeout-wrapped `connect` with a `--connect-timeout` flag, and a `#[cfg(unix)]`-gated import.

Wave note: this plan shares `crates/nosh-client/src/client.rs` and `src/main.rs` with Plan 09-01 (different regions, same files), so it runs in Wave 2 after 09-01 to keep file ownership exclusive.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authorized_keys file → server authorization set | Determines which client identities are trusted; a parse change must not widen the trusted set |
| network → client connect | An unresponsive/hostile endpoint must not hang the client indefinitely |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Elevation of Privilege | load_authorized_keys warn+skip | mitigate | Skip applies ONLY to lines from_openssh_line already rejects; the accepted-key set is a subset of (identical to) prior behavior for parseable lines. A malformed line is dropped, never coerced into an accepted key — fail-closed. Unit test asserts a mixed file yields exactly the Ed25519 key and nothing else |
| T-09-05 | Denial of Service | client connect | mitigate | tokio::time::timeout bounds the establishment await (default 10s); a dead/black-hole server no longer hangs the client. Bounded, configurable, error is explicit |
| T-09-06 | Information Disclosure | warn log of skipped entry | mitigate | Log only the parse error + key-type token, not full sensitive payload; authorized_keys holds public-key data but avoid dumping raw blobs into logs as a habit |
| T-09-SC | Tampering | cargo dependency changes | mitigate | Only tempfile (already used elsewhere in workspace dev-deps) may be added as a dev-dependency; no new runtime crates. slopcheck if any new crate is introduced |
