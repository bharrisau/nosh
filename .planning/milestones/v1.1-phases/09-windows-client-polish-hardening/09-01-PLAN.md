<objective>
Make the native Windows console usable for full-screen TUI programs (vim, less, arrows, PageUp/Down) and route Ctrl-C to the remote shell as a raw 0x03 byte instead of killing the client. Because Ctrl-C no longer quits the client, add the OpenSSH-style `~.` local-quit escape (on all platforms) so the user always has a clean local exit.

Purpose: Closes Phase 8's PARTIAL D-02 (raw-key/TUI input did not work on Windows) and provides the local-quit mechanism that the VT-input fix removes.
Output: Extended `RawModeGuard` with Windows console-mode handling, a `windows-sys` target dep, and an `EscapeState` machine in the stdin pump with usage docs.
</objective>

    Add an `EscapeState` type (small enum or struct) in main.rs implementing the OpenSSH client escape state machine. Track whether the stream is at line-start. Session start counts as line-start. After a `\n` byte is forwarded, the next byte is at line-start. Behavior, processing the local stdin buffer byte-by-byte:
    - At line-start, a `~` byte does NOT get forwarded yet; transition to "seen-tilde".
    - In "seen-tilde": a `.` triggers local quit (return a signal to the caller, forward nothing); a `~` forwards a single literal `~` and returns to mid-line (NOT line-start â€” a literal tilde is not a newline); any other byte forwards BOTH the pending `~` and that byte, then updates line-start based on whether that byte was `\n`.
    - A `~` that is NOT at line-start is forwarded literally with no escape semantics.

    Implement `EscapeState::process(&mut self, input: &[u8]) -> EscapeAction` (or equivalent) that returns the bytes to forward to the server plus a flag indicating local quit was requested.
