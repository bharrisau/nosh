<objective>
Write the headless connection-migration validation test (the heart of Phase 7) and the human
live-check doc.

The test opens a real authenticated PTY session, drives a long-running monotonically-numbered
output stream, forces a client path change via `Endpoint::rebind()` onto a fresh loopback socket
mid-stream, and proves the SAME QUIC connection continues with no loss, no reorder, and no
connection error (D-03). It measures and logs the post-migration anti-amplification stall as a
multiple of RTT, soft-warning above ~3x but not failing (D-04). It produces a qlog artifact and
asserts it parses, while binding the CID-rotation / path-validation proof to `Connection::stats()`
FrameStats deltas (D-05), because quinn 0.11.9's qlog does not record frames or CIDs.

It also writes a short operator doc for the real Wi-Fi→cellular check (D-06), which is
non-blocking: Phase 7 completes autonomously and the operator records the live PASS later.

This plan adds NO production code — it is the validation deliverable on top of Plan 01's
explicit migration flag + qlog/rebind harness tooling.
</objective>

  <name>Task 1: Headless migration test — rebind mid-stream, assert same-connection continuity, measure stall, assert CID rotation, validate qlog</name>
  <files>crates/nosh-client/tests/migration.rs</files>
  <read_first>crates/nosh-client/tests/session.rs, crates/nosh-client/tests/common/mod.rs, crates/nosh-client/src/client.rs, .planning/phases/07-connection-migration-validation/07-RESEARCH.md, .planning/research/PITFALLS.md</read_first>
  <action>Create `crates/nosh-client/tests/migration.rs` with `mod common;` and a single `#[tokio::test]` (e.g. `migration_survives_path_change`), guarded by a `have_sh()` check on `/bin/sh` that returns early (mirroring tests/session.rs). Wrap the whole body in a generous `tokio::time::timeout(Duration::from_secs(30), ...)` so a hang fails loudly. Steps:
1. Generate `TestKey`s; spawn the server with `spawn_server_with_shell(&host_key, &[&client_key.public], AuthLimits::default(), Some("/bin/sh".into()))`. (build_server_config now sets migration(true) — Plan 01.)
2. Create a temp dir; build a qlog path (`dir/migration.qlog`) and an empty known_hosts; build the client endpoint via `common::client_endpoint_with_qlog(client_key.client_identity(), known_hosts, &qlog_path)`. `client::connect(&endpoint, server.addr, HOST)`.
3. `client::open_session(&conn, "xterm", 80, 24, vec![])` → (send, recv). (run_session_collect won't work here — you must interleave reading with the rebind, so drive the streams manually.) Discard the initial `SessionOpened` token frame if present (open_session does not, but run_session_collect/open_session_with_token do — read frames until the first PtyData, ignoring non-PtyData control frames, OR use open_session and tolerate a leading control frame).
4. Drive a long monotonic stream: `client::send_input(&mut send, b"i=0; while [ $i -lt 80 ]; do echo LINE:$i; i=$((i+1)); sleep 0.05; done; echo DONE; exit 0\n")`. This yields LINE:0..LINE:79 over ~4s — comfortably spanning the rebind + sub-ms loopback path validation.
5. Read PtyData frames, parsing `LINE:<n>` integers in order into a Vec. After observing a mid-stream marker (e.g. once you have seen LINE:10), capture the PRE-rebind snapshot: `let pre = conn.stats();`, `let pre_addr = conn.remote_address();`, `let pre_id = conn.stable_id();`, `let rtt = conn.rtt().max(conn.stats().path.rtt);` and `let t_last_pre = Instant::now()` updated on every frame so it holds the time of the last frame seen before the rebind.
6. Force the path change: `let new_addr = common::rebind_client(&endpoint).expect("rebind");`. (Optional Pitfall #2 mitigation: immediately `send_input(&mut send, b"")`/a tiny PtyData to advance the server anti-amplification budget — do this BEFORE continuing reads.) Record `t_rebind = Instant::now()`.
7. Continue reading frames until `DONE`/`SessionClose`. On the FIRST frame received strictly after the rebind, record `t_first_post = Instant::now()` and compute the measured stall `t_first_post - t_last_pre`. HARD-FAIL (panic/assert) on any `read_message` error that is a `ConnectionError`, or if `conn.close_reason()` becomes a transport (non-application) error during the read loop.
8. After the stream ends, ASSERT CONTINUITY (D-03): the parsed sequence is exactly 0,1,2,...,79 with NO gaps, NO duplicates, NO reordering (a single strictly-increasing run covering every value). ASSERT SAME CONNECTION: `conn.stable_id() == pre_id` (the connection object never changed — there is only one TLS handshake per quinn Connection; this proves no new handshake). It is OK (and expected after a successful migration) that `conn.remote_address() != pre_addr`; do not require it to change (loopback may keep the addr stable in some cases) — log it.
9. MEASURE + LOG THE STALL (D-04): compute `ratio = measured_stall.as_secs_f64() / rtt.as_secs_f64().max(1e-6)`; `eprintln!`/`tracing::info!` the measured stall and ratio (e.g. "migration stall: {:?} (~{:.1}x RTT)"). If `ratio > 3.0`, emit a SOFT `eprintln!`/`tracing::warn!` ("anti-amplification stall exceeded 3x RTT (expected RFC 9000 §9.4 behavior)") but DO NOT assert/fail on it.
10. ASSERT CID ROTATION / PATH VALIDATION (D-05 binding proof): `let post = conn.stats();`. ASSERT `post.frame_tx.path_challenge + post.frame_rx.path_challenge > pre.frame_tx.path_challenge + pre.frame_rx.path_challenge` (path validation ran across the rebind) AND `(post.frame_tx.new_connection_id + post.frame_tx.retire_connection_id + post.frame_rx.new_connection_id + post.frame_rx.retire_connection_id) > (pre...same sum)` (CIDs rotated — RFC 9000 §9.5). Add a comment explaining these counters are the version-stable proof because quinn 0.11.9 qlog lacks frame/CID detail.
11. VALIDATE QLOG ARTIFACT (D-05 literal): drop/close the connection (`conn.close(0u32.into(), b"done"); endpoint.close(...)`) so the qlog streamer finalizes and flushes, give it a brief settle (`sleep(100ms)`), then assert the qlog file at `qlog_path` exists and is non-empty, and that it parses (e.g. read to string; assert the first non-empty line is valid JSON via the `qlog` crate types OR a minimal check that it begins with `{` and the file is well-formed JSON-seq — choose the lightest check that does not require adding serde_json unless serde_json is already in the build graph; a "file exists, len > 0, first byte is '{' or a record-separator" smoke check is acceptable since the binding CID proof is the FrameStats assertion). Comment that quinn qlog does not record PATH_CHALLENGE frames so this is an artifact-presence check, not the CID assertion.
Keep timeouts on every read (`tokio::time::timeout(Duration::from_secs(10), read_message(..))`) so a stuck read fails rather than hangs.</action>
