<objective>
Wire the Plan-01 registry into the connection pump in `crates/nosh-server/src/server.rs`. Three
changes: (1) create one `Arc<SessionRegistry>` in `run_accept_loop`, spawn its reaper once, and
clone the registry into each connection task; (2) on session open, build a `SessionSlot`, register
it Active, and feed every outgoing PTY chunk into the slot's `SequencedOutputBuffer`; (3)
SUBDIVIDE the disconnect outcome (D-02) so a transport-level loss ORPHANS the slot (keep MasterPty
open, NO SIGHUP — Pitfall #7) while an explicit `SessionClose` or normal shell exit tears down
immediately (current behavior, D-01).

Purpose: This is the behavioral heart of the phase — making a session survive a dropped
connection instead of being killed. Get the orphan-vs-teardown split exactly right.
Output: Updated `server.rs`; new integration tests in `crates/nosh-client/tests/persistence.rs`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client QUIC connection ↔ server session | A dropped/abruptly-closed connection is untrusted input that must transition the session to Orphaned, not crash or leak the pump tasks. |
| client-signalled SessionClose ↔ teardown | A SessionClose is an in-band client request to end the session; it must tear down (not orphan) so a malicious client cannot force unbounded orphan accumulation by sending SessionClose-then-reconnect... (covered by Plan 01 cap). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Denial of Service | orphan-on-disconnect path | mitigate | Orphaning routes through the Plan-01 per-identity cap (orphan() enforces it); a reconnect-loop client cannot grow orphans past the cap. |
| T-05-06 | Tampering | resize / input on orphaned session | accept | After transport loss the QUIC stream is dead; no client input reaches the orphaned PTY until Phase 6 reattach re-authenticates (TLS mutual auth, unchanged). |
| T-05-07 | Repudiation | session lifecycle logging | mitigate | tracing span (server.rs:239) already records session_id + identity fingerprint; orphan/teardown transitions are logged for audit. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new crates; tests reuse existing dev-deps (tempfile, tokio). |
</threat_model>
