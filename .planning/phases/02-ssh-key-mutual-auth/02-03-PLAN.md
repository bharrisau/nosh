# Plan 02-03: Endpoint wiring and DoS hardening

## Objective
Swap the Phase 1 placeholder wiring for real SSH-key mutual auth: server uses
its host key cert + `AuthorizedKeysVerifier`; client uses the agent-signed
identity cert + `HostKeyVerifier`. Add the pre-auth concurrency cap and auth
timeout to the accept loop. Covers AUTH-01/02/04/05 at the integration seam.

## must_haves
- truths:
  - "Server moves off with_no_client_auth() to with_client_cert_verifier (D-10)"
  - "Client presents an agent-signed Ed25519 identity cert (AUTH-04)"
  - "Accept loop bounds concurrent half-open conns (Semaphore, default 64) and times out auth (default 5s) (AUTH-05, D-13)"
  - "Server host key + authorized_keys + client identity + known_hosts paths are flag-overridable (D-08)"

<action>
Add an `AuthLimits { max_concurrent: usize, auth_timeout: Duration }` (defaults 64 / 5s) and
thread it into `run_accept_loop`. Before `endpoint.accept().await` handling, acquire a permit
from a `tokio::sync::Semaphore` (capacity `max_concurrent`); inside the per-connection task wrap
`incoming.await` in `tokio::time::timeout(auth_timeout, ...)`, dropping the connection (and
permit) on elapse. The permit is held until the handshake completes/fails then released. Log a
warn on timeout.
