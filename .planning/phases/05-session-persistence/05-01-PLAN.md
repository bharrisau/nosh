<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client identity → registry key | The SSH identity (proven by TLS mutual auth in Phase 2/4) is the cap-enforcement and slot-ownership key. A wrong key must never index another identity's slots. |
| orphaned PTY/shell ↔ server memory | Orphaned sessions hold a live shell + buffered output; unbounded growth is a DoS vector. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Denial of Service | orphan store (per-identity) | mitigate | Per-identity cap (default 5, D-05) enforced before the first orphan is stored; LRU eviction of the oldest orphan with a logged warning (Pitfall #5). |
| T-05-02 | Denial of Service | shell child reaping | mitigate | Background reaper try_wait()s orphans so exited shells never accumulate as zombies (Pitfall #6). |
| T-05-03 | Elevation of Privilege | registry identity key | mitigate | HashMap keyed by the authenticated NoshPublicKey raw bytes; eviction/orphan operate only within the matching identity's Vec — no cross-identity access. |
| T-05-04 | Information disclosure | private key / identity in logs | accept | Logs use the public SHA256 fingerprint only (never private material); fingerprint is non-secret (mirrors ssh-keygen -l). |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new crates introduced (VecDeque/Bytes/uuid/nix all already present) — no package-install surface this plan. |
</threat_model>
