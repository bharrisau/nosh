<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stdin → run_pump → send_input | Local keystrokes forwarded verbatim to the server reliable stream; the predictor reads a copy for display only. |
| server datagram → cull → render | Server-confirmed grid + epoch is the display authority; predictions reconcile against it. |
| predictor → stdout (render_with_predictor) | The single display path; must not leak unechoed characters and must never become a second writer. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-05 | Information disclosure | password chars rendered via prediction during `read -s` | mitigate | Engine keeps predictions tentative (confirmed_epoch frozen) when the server does not echo; `render_with_predictor` consults `predictor.cell_at` which returns None for tentative cells → zero predicted password chars. Validated adversarially in Plan 03 (PREDICT-04 / D-15-01c). |
| T-15-06 | Tampering | prediction altering bytes sent to the server | mitigate | `predictor.on_input` is called AFTER `escape.process` and BEFORE `send_input`, and `send_input` still forwards `result.bytes_to_forward` verbatim — the predictor receives a borrow and cannot mutate the forwarded slice. Acceptance grep asserts the unchanged send call. |
| T-15-07 | Tampering | second display path bypassing the single-render seam | mitigate | All speculative output goes through `ClientScreen::render_with_predictor` (the existing render path extended). No new `stdout.write_all` outside the established buffered-flush pattern; CLAUDE.md single-display-path invariant preserved. Verified: no direct ANSI emission added outside screen.rs. |
| T-15-08 | Information disclosure | latency instrumentation logging keystroke content | mitigate | The Phase-17 instrumentation hook logs only timing + epoch numbers (no character content) at debug level under target `nosh::predict`; off by default. Hook records `latency_ms`/`epoch`, never `ch`. |
| T-15-09 | Denial of service | per-keystroke full re-render cost on bulk/paste | accept | Bulk/paste suppression in the engine (Plan 01) prevents per-char prediction floods; the render is the same minimal-diff path already shipped in Phase 14. Low risk on a single interactive session. |
</threat_model>
