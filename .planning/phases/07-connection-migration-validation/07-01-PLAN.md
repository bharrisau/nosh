<objective>
Make the two production-adjacent changes Phase 7 needs, plus the qlog test infrastructure the
headless migration test (Plan 02) depends on:

1. **D-01:** set `ServerConfig::migration(true)` explicitly in `build_server_config`, with a
   comment documenting that this is deliberate (not relying on the quinn default) so roaming
   cannot be silently disabled by a future default change.
2. **D-05 (mechanism):** enable quinn's `qlog` cargo feature and add a test-harness path that
   builds a client `quinn::Endpoint` whose `TransportConfig` carries a `QlogStream` writing to a
   caller-supplied file, plus a small helper to `rebind()` the client endpoint onto a fresh
   loopback UDP socket.
3. **Pitfall #4:** confirm (and comment) that the existing keep-alive/idle-timeout settings in
   `transport.rs` survive migration — do NOT change them.

Purpose: isolate the build-system + production change (one config line + a feature flag + an
additive test constructor) into a Wave 1 plan so the actual migration test (Plan 02) can be
written against a stable, green workspace. This plan adds NO new assertions itself — it makes
the migration path explicit and gives Plan 02 the tools (qlog endpoint + rebind helper).
</objective>
