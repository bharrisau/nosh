`fn compose_desired(&self) -> Vec<Vec<Cell>>`: clone `self.confirmed` into `desired`; for each `overlay` in `self.overlays`, for each (row, col) within bounds, if `overlay.cell_at(row as u16, col as u16)` is `Some(cell)` overwrite `desired[row][col]`. `pub fn render_to_stdout<W: Write>(&mut self, out: &mut W) -> std::io::Result<()>` (Mosh Display model): compute `desired = self.compose_desired()` and `desired_cursor = self.confirmed_cursor`; iterate `rows = desired.len().min(self.physical.len())`, `cols = min` of row widths; track `last_row/last_col: Option<u16>` and `last_sgr: Option<(CellStyle, Option<u8>, Option<u8>)>`. For each cell where `desired[r][c] != physical[r][c]`: if not already at (r,c) emit `out.queue(MoveTo(col as u16, row as u16))?` (Pitfall 7 — MoveTo is (col,row), 0-based) and update last_row/last_col; if `last_sgr != Some(want_sgr)` call `emit_sgr` and update last_sgr; write the char via `ch.encode_utf8`; advance `last_col`. After the loop, `out.queue(MoveTo(desired_cursor.col, desired_cursor.row))?` and `out.flush()?`. Then set `physical = desired` cell-by-cell and `self.physical_cursor = desired_cursor`. This is the ONLY function that writes display output (CLAUDE.md single-path invariant) — do not add any other stdout writer.

`fn emit_sgr<W: Write>(out, style, fg, bg) -> std::io::Result<()>`: build `params = String::from("0")`; push `";1"`/`";3"`/`";4"`/`";7"` per the BOLD/ITALIC/UNDERLINE/REVERSE bits; push `";38;5;{n}"` for `Some(n)` fg and `";48;5;{n}"` for `Some(n)` bg; `write!(out, "\x1b[{params}m")`.

`pub fn reset_physical(&mut self)`: `self.physical = Self::make_grid(self.cols, self.rows); self.physical_cursor = CursorPos { row: 0, col: 0 };` (used by the reattach path in Plan 02 to force a full repaint).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server → client datagram | The server (possibly compromised/malicious) sends StateDiff datagrams that the client decodes and applies to its grid. Untrusted input crosses here. |
| client grid → local stdout | Composed cells are emitted as ANSI to the local terminal. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering | `ClientScreen::apply` (out-of-bounds row/col in a crafted DiffRun) | mitigate | `run.row >= confirmed.len()` → continue; `col >= row_cells.len()` → break. No panic, no OOB write. Unit test covers an over-range run. |
| T-14-02 | Denial of Service | `ClientScreen::apply` (oversized run count / huge dimensions) | mitigate | `decode_datagram` already caps runs at `MAX_RUNS=4096` (Phase 11); confirmed grid is bounded by `cols × rows`. apply allocates nothing per-run beyond the bounded grid. |
| T-14-03 | Tampering | `ClientScreen::apply` (stale/replayed old-epoch datagram) | mitigate | D-14-05 monotonic guard: `if diff.epoch <= last_applied_epoch { return; }` — rollback/replay of an older confirmed grid is rejected. |
| T-14-04 | Tampering | `render_to_stdout` (attacker-controlled bytes injected to terminal) | mitigate | Only `Cell.ch` (a single validated Unicode scalar from the server's parsed TerminalState) is written per changed cell, plus client-generated MoveTo/SGR. No raw server byte stream reaches stdout (the PtyData→stdout path is removed in Plan 02). No terminal-injection vector beyond what the remote shell itself can already produce. |
| T-14-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies in Phase 14 — all crates already in workspace Cargo.toml (RESEARCH Package Legitimacy Audit). crossterm flagged [SUS] is a confirmed false positive (name proximity to crossbeam). No install task; no blocking checkpoint required. |
