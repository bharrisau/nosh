## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server → client datagram channel | `conn.read_datagram()` delivers attacker-influenceable StateDiff bytes into the client pump. |
| client → local stdout | Composed ANSI from `render_to_stdout` is the only thing written for display. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-05 | Tampering | datagram arm decode (malformed/oversized datagram) | mitigate | `decode_datagram` validates framing and caps runs at MAX_RUNS (Phase 11); a decode error is silently discarded (no apply, no render). ClientScreen.apply (Plan 01) bounds-checks rows/cols. |
| T-14-06 | Tampering | datagram arm (stale-epoch replay forcing display rollback) | mitigate | `if diff.epoch > screen.last_applied_epoch()` gate in the arm + D-14-05 guard inside apply — an old datagram never rolls the confirmed display back. |
| T-14-07 | Information Disclosure | epoch-ack on wrong channel | mitigate | epoch-ack sent via `conn.send_datagram(encode_epoch_ack(..))` (TAG_CLIENT_EPOCH datagram), kept strictly separate from the reliable-stream `Ack{seq}` (Pitfall 3); no session/credential data is in the ack (just the u64 epoch). |
| T-14-08 | Tampering | terminal injection via PtyData→stdout | mitigate | The `stdout.write_all(&data)` PtyData display path is REMOVED. Only client-composed cells (validated Unicode scalars) + client MoveTo/SGR reach stdout. Verified by grep gate: PtyData arm has zero stdout writes. |
| T-14-DoS | Denial of Service | epoch-ack flood / best-effort send failure | accept | `send_datagram` is best-effort; a failed send (queue full) is ignored and the server falls back to full-screen diffs next tick (RESEARCH A6). No session loss; low value. |
| T-14-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies in Phase 14 (RESEARCH Package Legitimacy Audit). No install task; no blocking checkpoint required. |
