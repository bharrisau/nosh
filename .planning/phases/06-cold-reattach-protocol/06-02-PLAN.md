<objective>
Extend the Phase 5 `SessionRegistry`/`SessionSlot`/`SequencedOutputBuffer` (in `registry.rs`) with everything reattach authorization and output continuity need — WITHOUT rebuilding Phase 5. Add: a per-slot single-use token, a `Reconnecting` lifecycle state, an atomic two-factor `reattach()` lookup with a uniform opaque rejection, output `replay_from` and continuous-ack `trim_acked`. Cover it all with unit tests including the no-oracle and mutual-exclusion paths.
</objective>

<context>
This plan is purely additive to `crates/nosh-server/src/registry.rs`. The existing types and their invariants (read them first) must be preserved exactly:
- `SessionRegistry.inner: Mutex<HashMap<[u8;32], Vec<Arc<SessionSlot>>>>` — keyed by SSH identity's raw key32. KEEP this keying. The token is looked up by scanning the per-identity Vec (n <= cap = 5), mirroring the existing O(n) scans in `orphan()`.
- `SessionSlot` is held as `Arc` and removed by `Arc::ptr_eq` in `remove_slot` (instance identity, not session_id). Reattach MUST resume the existing `Arc` instance.
- `SequencedOutputBuffer`: monotonic `next_seq` (starts 0), `ring: VecDeque<(u64, Bytes)>`, `total_bytes`/`max_bytes` (64 KiB), `lowest_retained_seq`, `truncated`. `push()` enforces the cap (drop-oldest, newest always survives) and sets `truncated`/`lowest_retained_seq` on cap eviction.

ARCHITECTURE.md sketches keying the registry by token — do NOT follow that; CONTEXT.md reconciles it to identity-primary + token-selector.
</context>

<task name="Add SequencedOutputBuffer::replay_from and trim_acked">
<read_first>
- crates/nosh-server/src/registry.rs (SequencedOutputBuffer: ring, total_bytes, lowest_retained_seq, truncated, push)
</read_first>
<action>
Add two methods to `SequencedOutputBuffer`:

`pub fn replay_from(&self, last_acked_seq: u64) -> (Vec<(u64, bytes::Bytes)>, u64, bool)` — returns (chunks_to_replay, replaying_from_seq, truncated_below_request). Honor the `Reattach.last_acked_seq` convention from plan 06-01: replay all ring entries with `seq > last_acked_seq`. Determine `truncated_below_request`: if the ring is non-empty and its front seq is greater than `last_acked_seq + 1` (the client's requested resume point fell out of the ring due to the cap), set it true and `replaying_from_seq` = front ring seq; otherwise false and `replaying_from_seq` = `last_acked_seq + 1` (or the front seq if the ring starts later but contiguous — pick the first replayed seq). Return the chunks as owned `(u64, Bytes)` clones (Bytes clone is cheap/refcounted). The chunks must be in ascending seq order with no gaps relative to what the ring holds (Pitfall #9: no dup, no gap within retained range).

`pub fn trim_acked(&mut self, acked_seq: u64)` — drop ring entries with `seq <= acked_seq`, decrementing `total_bytes` by each dropped chunk's length (D-08 continuous ack). This is NOT a data-loss truncation (the client already has those bytes), so it must NOT set `self.truncated` and must NOT advance `lowest_retained_seq` in a way that signals data loss — only the cap-eviction path in `push()` sets `truncated`. (Acked bytes leaving the ring is normal; if you update `lowest_retained_seq` to the new front for bookkeeping, do NOT set `truncated`.)
</action>
<acceptance_criteria>
- `replay_from(N)` returns only chunks with seq > N, ascending, no duplicates
- When the requested resume point is below `lowest_retained_seq` (cap dropped it), `replay_from` returns `truncated_below_request == true`
- `trim_acked(N)` removes chunks with seq <= N and reduces `len_bytes()` accordingly, and does NOT set `truncated()`
- `cargo build -p nosh-server` succeeds
</acceptance_criteria>
</task>

<task name="Implement SessionRegistry::reattach (two-factor, atomic, opaque)">
<read_first>
- crates/nosh-server/src/registry.rs (SessionRegistry::orphan for the lock+per-identity-Vec-scan pattern; NoshPublicKey::key32/fingerprint usage)
- .planning/research/PITFALLS.md (#8 two-factor, #10 race)
</read_first>
<action>
Add a private rejection enum used internally only (the server collapses it to a single wire `ReattachErr`): `enum ReattachReject { NotFound, IdentityMismatch, NotOrphaned }` (or similar). The PUBLIC method returns `Result<Arc<SessionSlot>, ReattachReject>` so the server can map every variant to the SAME opaque `Message::ReattachErr`. (Keep the granular enum only for internal tracing of an OUTCOME label, never the token.)

`pub fn reattach(&self, token: &[u8; 16], identity: &NoshPublicKey) -> Result<Arc<SessionSlot>, ReattachReject>`:
1. Acquire `self.inner.lock()` ONCE for the whole lookup+transition (atomicity, D-12 / Pitfall #10).
2. Scope to `slots` for `identity.key32()`. If the identity has no entry → `Err(NotFound)`.
3. Find a slot in that Vec whose `token() == *token`. If none → `Err(NotFound)`. (Because we scoped to the identity first, a valid token presented under a DIFFERENT identity simply won't be found in that identity's Vec → same `NotFound` path → same opaque error. This is the no-oracle property; there is no separate "identity mismatch reveals token exists" branch.)
4. Defense-in-depth identity check: assert the found slot's `identity == *identity` (it must, since it lives in this identity's Vec); if somehow not, treat as `Err(IdentityMismatch)`. (Belt-and-suspenders for D-06.)
5. Inspect the slot state: only `Orphaned` is reattachable. If `Active` or `Reconnecting` → `Err(NotOrphaned)` (D-12 mutual exclusion — blocks the two-clients-one-session race).
6. On success: transition the slot `Orphaned -> Reconnecting` (call `slot.mark_reconnecting()`) WHILE STILL HOLDING THE LOCK, then `Ok(slot.clone())` (clone the existing Arc — do NOT build a new slot). Drop the lock after returning the clone.

Logging: on success log `tracing::info!(identity = %identity.fingerprint(), "reattach accepted")`; on rejection log `tracing::info!(identity = %identity.fingerprint(), "reattach rejected")` with NO token, NO reject-variant detail that could distinguish causes in a way an attacker reads (the internal variant may be logged at debug for operators, but the WIRE response is always the single opaque `ReattachErr`). Never log the token.

Note: the server (next plan) is responsible for calling `slot.mark_active()` after replay completes and for `slot.rotate_token()` to mint the `ReattachOk.new_token`.
</action>
<acceptance_criteria>
- `SessionRegistry::reattach(&[u8;16], &NoshPublicKey)` exists returning `Result<Arc<SessionSlot>, _>`
- All failure causes are reachable and the server can map every Err variant to one opaque ReattachErr (the method does not itself emit wire frames)
- On success the slot transitions to `Reconnecting` and the returned Arc is the SAME instance (Arc::ptr_eq with the registry's stored slot)
- No `tracing` call in `reattach` references the token
- `cargo build -p nosh-server` succeeds
</acceptance_criteria>
</task>
