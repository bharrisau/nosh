<task>
<objective>Add session-driver library functions (raw-mode guard, env collection, headless driver).</objective>
<read_first>
- crates/nosh-client/src/client.rs (existing connect + echo helpers; keep connect/build_*)
- crates/nosh-proto/src/codec.rs (read_message/write_message)

<action>
In `crates/nosh-client/src/client.rs` add (keep existing functions; the echo helpers may stay
for the transport tests):
- `pub struct RawModeGuard;` with `pub fn enable() -> std::io::Result<RawModeGuard>` calling
  `crossterm::terminal::enable_raw_mode()`, and `impl Drop` calling
  `let _ = crossterm::terminal::disable_raw_mode();`. Document that Drop runs on normal return,
  unwind/panic, and the abrupt-network-loss error path (guard held in the run scope) — but NOT
  on SIGKILL (human-verify item).
- `pub fn collect_client_env() -> Vec<(String,String)>` returning the process's `TERM` plus any
  `LANG`, `TZ`, and `LC_*` vars (the SendEnv-style whitelist, D-05). Never include
  `SSH_AUTH_SOCK` or `LD_*`.
- `pub async fn open_session(conn: &quinn::Connection, term: String, cols: u16, rows: u16,
  env: Vec<(String,String)>) -> anyhow::Result<(quinn::SendStream, quinn::RecvStream)>` that
  `open_bi()`, `write_message(SessionOpen{..})`, and returns the stream halves.
- `pub async fn send_input(send, bytes)` → `write_message(PtyData{data})`.
- `pub async fn send_resize(send, cols, rows)` → `write_message(Resize{..})`.
- `pub async fn run_session_collect(conn, open_args, input_script: &[u8]) -> anyhow::Result<(Vec<u8>, i32)>`:
  a HEADLESS driver for tests — opens the session, writes `input_script` as one or more
  `PtyData` frames, then reads frames in a loop appending `PtyData` payloads to an output buffer
  until `SessionClose{exit_code,..}` is received (or stream closes), returning
  `(collected_output, exit_code)`. This is the seam the integration tests use without a tty.
</action>

</acceptance_criteria>
</task>

<task>
<objective>Rewrite the client binary main into an interactive session with raw mode + SIGWINCH + exit code.</objective>
<read_first>
- crates/nosh-client/src/main.rs (current echo-based main)
- crates/nosh-client/src/client.rs (the new session helpers)
</read_first>
<action>
Rewrite `main.rs` after `connect(..)`:
- Determine initial terminal size via `crossterm::terminal::size()` (fallback 80x24).
- `let _guard = client::RawModeGuard::enable()?;` held for the session (RAII restore — SESS-03).
- `open_session(&conn, term=$TERM or "xterm-256color", cols, rows, collect_client_env())`.
- Spawn the STDIN→stream pump: read raw stdin bytes (`tokio::io::stdin`) in a loop and
  `send_input` them as `PtyData` (Ctrl-C passes through as 0x03 — SESS-06; do NOT trap it).
- Spawn the stream→STDOUT pump: `read_message`; on `PtyData` write to `tokio::io::stdout` +
  flush; on `SessionClose{exit_code,reason}` record the code and break.
- SIGWINCH: `tokio::signal::unix::signal(SignalKind::window_change())`; on each signal arm a
  ~40ms debounce (`tokio::time::sleep`), and when it fires query `crossterm::terminal::size()`
  and `send_resize`. Coalesce bursts (only the latest size is sent).
- `tokio::select!` the pumps; when the session ends, drop the guard (restores terminal), close
  the connection cleanly (`conn.close(0u32.into(), b"client done")`), `endpoint.wait_idle()`,
  then `std::process::exit(exit_code)` so the client process surfaces the remote exit code
  (SESS-08). Remove the old echo/datagram round-trip calls and the idle-hold flag usage (the
  flag may be removed or repurposed; prefer removing it).
Keep the clap args for addr/port/host/identity/known_hosts; drop `idle_hold_secs`.
</action>
<acceptance_criteria>
- `main.rs` no longer calls `stream_echo_roundtrip`/`datagram_roundtrip`/`concurrent_roundtrip`
- main enables raw mode via the guard, opens a session, pumps stdin/stdout, handles SIGWINCH,
  and exits with the SessionClose exit code
- `cargo build -p nosh-client` succeeds; `cargo clippy -p nosh-client --all-targets` clean
</acceptance_criteria>
</task>

<action>
Create `crates/nosh-client/tests/session.rs` (uses `mod common;`). Update the harness if needed
so `spawn_server` can pass a `shell_override` (add a `spawn_server_with_shell(..., Option<String>)`
in `common/mod.rs` that threads it into `run_accept_loop`; keep the old signature delegating with
`None`). All session tests force `--shell /bin/sh` for portability and skip (return early with an
eprintln) if `/bin/sh` is absent.

Tests (each: generate keys, spawn_server_with_shell(Some("/bin/sh")), connect via harness, then
use `client::run_session_collect` / `open_session`):
- `sess01_02_04_real_tty_and_io`: open cols=132 rows=40; input script
  `b"test -t 0 && echo IS_TTY; echo hello-nosh; stty size; exit 0\n"`; assert output contains
  `IS_TTY` (SESS-01 real tty), `hello-nosh` (SESS-02), and `40 132` (SESS-04 size).
- `sess05_resize`: open cols=80 rows=24; via `open_session`+manual frames: send a `Resize{cols:100,
  rows:50}`, then `PtyData(b"stty size; exit 0\n")`, collect; assert output contains `50 100`.
- `sess07_env_sanitization`: open with env including `("LD_PRELOAD","/evil.so")`,
  `("BASH_ENV","/x")`, `("SSH_AUTH_SOCK","/a")`, `("IFS","x")`, `("SHELLOPTS","y")`,
  `("LC_ALL","C")`, `("TZ","UTC")`, `("TERM","xterm-256color")`; input `b"env; exit 0\n"`;
  assert output CONTAINS `LC_ALL=C`, `TZ=UTC`, `TERM=xterm-256color`; assert output does NOT
  contain `LD_PRELOAD`, `BASH_ENV=`, `SSH_AUTH_SOCK`, `SHELLOPTS=` (explicit security assertions).
- `sess08_exit_code`: input `b"exit 42\n"`; assert the returned exit code == 42 (SESS-08).
- `sess09_clean_close`: run a trivial `exit 0` session; after it ends assert
  `conn.close_reason()` is an `ApplicationClose` (clean structured close, not a transport error).
- `sess10_no_zombie`: open a session running an interactive `cat` (a process that stays alive);
  capture the child pid by having the server log it OR by running `b"echo $$; sleep 30\n"` and
  parsing the printed PID from output, then DROP the connection (close the client endpoint /
  abort) and poll for up to ~3s that the pid is gone or in a non-Z reaped state. Use
  `nix::sys::wait`-free check: read `/proc/<pid>/stat` and assert the process either no longer
  exists OR its state is not `Z` after the server's SIGHUP+reap. (Add `nix` to client
  dev-deps if needed, or read /proc directly with std::fs.) Tolerate timing with a bounded poll.

Use `tokio::time::timeout` around session collection so a hung test fails fast.
</action>
