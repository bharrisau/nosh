# Plan 03-01: Extend the Message protocol for PTY sessions

Extend the existing `nosh-proto` `Message` enum (which already has `SessionClose`) with the
session-traffic variants. ALL session traffic is framed via this enum + the existing codec
(D-01). No codec wire-format change — postcard handles the new variants automatically; only
add a round-trip test for the new shapes.

<task>
<objective>Add SessionOpen, PtyData, Resize variants to Message.</objective>
<read_first>
- crates/nosh-proto/src/messages.rs (current enum — has only SessionClose)
- crates/nosh-proto/src/codec.rs (length-delimited postcard codec; do not change wire format)
</read_first>
<action>
In `crates/nosh-proto/src/messages.rs`, add three variants to `Message` (keep `SessionClose`
unchanged):
- `SessionOpen { term: String, cols: u16, rows: u16, env: Vec<(String, String)> }` — first
  frame the client sends; `env` is the client's forwarded/whitelisted vars (server re-filters,
  D-06). Use `Vec<(String,String)>` (NOT a map) for deterministic postcard ordering and easy
  test assertions.
- `PtyData { data: Vec<u8> }` — raw PTY bytes, both directions (keystrokes client→server,
  output server→client).
- `Resize { cols: u16, rows: u16 }` — window resize (SESS-05).
Update the module doc comment to describe the session frames. Keep `#[derive(Debug, Clone,
PartialEq, Eq, Serialize, Deserialize)]`.
</action>
<acceptance_criteria>
- `messages.rs` contains `SessionOpen {`, `PtyData {`, `Resize {`, and still `SessionClose {`
- `cargo build -p nosh-proto` succeeds
</acceptance_criteria>
</task>

<task>
<objective>Round-trip tests for the new variants through the codec.</objective>
<read_first>
- crates/nosh-proto/src/codec.rs (existing #[cfg(test)] mod tests for the pattern)
</read_first>
<action>
In `codec.rs` tests module, add a test `session_variants_round_trip` that, for each of
`SessionOpen` (with a non-empty `env` of at least 2 pairs, term="xterm-256color", cols=132,
rows=40), `PtyData { data: vec![0x03, b'l', b's', b'\n'] }`, and `Resize { cols: 100, rows: 50 }`,
calls `write_message` into a `Vec<u8>`, reads it back with `read_message` from a
`std::io::Cursor`, and asserts equality. Assert `SessionOpen.env` preserves order.
</action>
<acceptance_criteria>
- `cargo test -p nosh-proto` passes including `session_variants_round_trip`
- test asserts decoded `SessionOpen` equals the encoded one (env order preserved)
</acceptance_criteria>
</task>

<verification>
- `cargo build -p nosh-proto` and `cargo test -p nosh-proto` green.
- Every new variant survives a postcard round-trip through the length-delimited codec.
</verification>
