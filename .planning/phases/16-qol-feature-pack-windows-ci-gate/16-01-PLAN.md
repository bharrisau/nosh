<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| remote shell/PTY output → server TerminalState | Untrusted application bytes (incl. OSC sequences) cross into nosh's parser/state |
| server → client (reliable stream) | Server forwards selected control sequences to the client for out-of-band re-emit |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Information Disclosure | osc_dispatch OSC 52 read form | mitigate | `if data == b"?" { return; }` BEFORE storing osc52_pending — read/query form never stored, never forwarded (D-16-01a). Dedicated test `osc52_read_form_is_silently_dropped`. |
| T-16-02 | Denial of Service | osc_dispatch OSC accumulation (vte std re-enabled) | mitigate | Explicit caps `OSC_52_MAX_BYTES=65536`, `MAX_TITLE_BYTES=1024` in osc_dispatch; transient vte buffer bounded by OS pipe buffer. Adversarial large-OSC tests confirm bound + no panic (CR-03 regression guard). |
| T-16-03 | Tampering | OSC 0/2 title forwarded to client terminal | accept | Title is emitted by the local terminal as window-title text, not interpreted as markup/cells; no injection surface. Bounded to MAX_TITLE_BYTES. |
| T-16-04 | Tampering | proto enum discriminant reordering | mitigate | TerminalControl appended after Ack (append-only invariant, messages.rs lines 56-62); no existing discriminant shifts. |
| T-16-SC | Tampering | npm/pip/cargo installs | mitigate | No new external packages this plan — only the existing `vte` crate's `std` feature is re-enabled (no version change; legitimacy unchanged per RESEARCH Package Legitimacy Audit). No legitimacy checkpoint required. |
</threat_model>
