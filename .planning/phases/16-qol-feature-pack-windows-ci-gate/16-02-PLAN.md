## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server → client (reliable stream) | Client receives forwarded control sequences and re-emits them to its own stdout/terminal |
| client → local terminal emulator | Re-emitted OSC sequences are interpreted by the user's terminal (clipboard write, title) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-05 | Information Disclosure | client OSC 52 re-emit | mitigate | Client only ever re-emits Clipboard WRITE payloads (the read/query form was dropped server-side in Plan 16-01, T-16-01); the client never originates an OSC 52 read. Write-only by construction. |
| T-16-06 | Tampering | OSC re-emit interleaving with compositor render | accept | OSC 52/0/2 carry no cursor motion and do not write cells; tokio::select! arms serialize so no byte-level interleave (Pitfall 4). Out-of-band by design (D-16-01). |
| T-16-07 | Elevation of Privilege | WSAEMSGSIZE filter masking genuine errors | mitigate | Filter is `quinn_udp=error`, NOT `quinn=error` — only quinn_udp WARN is suppressed; quinn connection/auth WARNs remain visible (Security Domain table, RESEARCH). |
| T-16-08 | Information Disclosure | --status RTT title overwrites remote title | accept | User opts into --status; documented precedence (status title suppresses forwarded title, Pitfall 5). No sensitive data — RTT only. |
| T-16-SC | Tampering | npm/pip/cargo installs | mitigate | No new external packages — only the existing crossterm crate's `osc52` feature is enabled (no version change; legitimacy unchanged per RESEARCH Package Legitimacy Audit). No legitimacy checkpoint required. |
