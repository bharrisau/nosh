shared sparse terminal-diff contract every subsequent server (Phase 12/13) and
client (Phase 14/15) component depends on (SYNC-01).

(Phase 12), or render anything client-side (Phase 14/15).

one module — mirroring the `codec.rs` "wire format behind one module" convention
(D-03/D-04) so a future codec swap stays localized.

inline round-trip, decode-hardening, size-cap, cursor-priority, and
heterogeneous-fill (continue-past-rejection) tests.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| network → `decode_datagram` | Datagram bytes arrive from `conn.read_datagram()`. QUIC TLS 1.3 (RFC 9001) authenticates and decrypts each packet, so bytes cannot be injected from outside the session — but they may be malformed, truncated, or adversarially shaped at the application (postcard) layer. |
| `encode_datagram` output → wire | The function must never emit a payload that meets or exceeds the caller's `cap` (strict `payload.len() < cap`), or quinn's `send_datagram` would reject/truncate it downstream. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Denial of Service | `decode_datagram` on a malformed postcard body | mitigate | `postcard::from_bytes` returns `Err`, never panics; decode path has no `unwrap`/`expect`/`panic!` (Task B grep gate); error surfaces as `ProtoError::Postcard`. |
| T-11-02 | Denial of Service | `decode_datagram` with an oversized declared run-vector length | mitigate | Enforce `MAX_RUNS` (4096) sanity cap after decode; reject with `ProtoError` before the value is returned, bounding allocation. **Verification-backed by a dedicated MAX_RUNS test (Task B): a > MAX_RUNS payload must decode to Err, no panic.** |
| T-11-03 | Denial of Service | `decode_datagram` on empty or truncated input | mitigate | `split_first` on empty bytes returns `Err`; truncated postcard body returns `Err` — both produce `ProtoError`, no panic (Task B negative tests). |
