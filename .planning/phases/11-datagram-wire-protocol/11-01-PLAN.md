postcard 1.1.3 API surface (VERIFIED in RESEARCH.md, `alloc` feature already enabled):
- `postcard::to_allocvec(&value) -> Result<Vec<u8>, postcard::Error>`
- `postcard::from_bytes(&[u8]) -> Result<T, postcard::Error>`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| network → `decode_datagram` | Datagram bytes arrive from `conn.read_datagram()`. QUIC TLS 1.3 (RFC 9001) authenticates and decrypts each packet, so bytes cannot be injected from outside the session — but they may be malformed, truncated, or adversarially shaped at the application (postcard) layer. |
| `encode_datagram` output → wire | The function must never emit a payload that exceeds the caller's `cap`, or quinn's `send_datagram` would reject/truncate it downstream. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Denial of Service | `decode_datagram` on a malformed postcard body | mitigate | `postcard::from_bytes` returns `Err`, never panics; decode path has no `unwrap`/`expect`/`panic!` (Task 2 grep gate); error surfaces as `ProtoError::Postcard`. |
| T-11-02 | Denial of Service | `decode_datagram` with an oversized declared run-vector length | mitigate | Enforce `MAX_RUNS` (4096) sanity cap after decode; reject with `ProtoError` before the value is returned, bounding allocation (Task 2). |
| T-11-03 | Denial of Service | `decode_datagram` on empty or truncated input | mitigate | `split_first` on empty bytes returns `Err`; truncated postcard body returns `Err` — both produce `ProtoError`, no panic (Task 2 negative tests). |
| T-11-04 | Tampering | Stale / reordered datagram applied after a newer one | accept (in this phase) / mitigate (downstream) | The monotonic `epoch:u64` is the staleness/replay defense: the applying consumer (Phase 14) applies a diff only if `epoch > last_applied_epoch`. Phase 11 provides the epoch field and documents the contract; QUIC TLS 1.3 already prevents off-session injection. The app-layer epoch check is enforced at the Phase 14 callsite. |
| T-11-05 | Tampering | Decoded run row/col indices exceeding declared dimensions (out-of-bounds when later applied to a grid) | accept | `StateDiff` carries `cols`/`rows`; the bounds check belongs at the apply site (Phase 14 renderer), which clamps/rejects runs outside the grid. Phase 11 is the wire format only and does not apply diffs; documenting the carried dimensions here is the enabling contract. No Phase 11 code dereferences these indices. |
| T-11-06 | Information Disclosure | Datagram payload contents | accept | Confidentiality is provided by QUIC TLS 1.3 at the transport layer; `StateDiff` carries terminal cell text but no secrets beyond what the shell already renders. No additional app-layer encryption. |
| T-11-SC | Tampering | npm/pip/cargo installs | mitigate | Phase 11 installs ZERO new packages — all deps (`postcard`, `serde`, `bytes`, `quinn`) are pre-existing workspace members (RESEARCH Package Legitimacy Audit: no [SLOP], no [SUS]). No legitimacy checkpoint required. |
</threat_model>
