# Phase 999.3 Plan 02: D-02 Typematic Content-Inspection Batch Classification — Summary

**One-liner:** Replace the flat `>4-byte → BulkSuppressed` rule with UTF-8 content inspection: a batch of entirely single-width printable chars (key-repeat / fast typing) is now predicted char-by-char via a new `PredictBatch` enum variant instead of triggering a BulkSuppressed reset.

## What Was Built

Single fix in `crates/nosh-client/src/predictor.rs` with TDD RED/GREEN cycle and adversarial fail-before/pass-after tests (D-06).

### Design Choice: Option A — `InputAction::PredictBatch(Vec<(char, u16)>)`

Added `PredictBatch(Vec<(char, u16)>)` to the `InputAction` enum. The variant carries each `(char, col_width)` pair for the chars in the batch. `on_input` handles it in a new match arm that iterates and applies the PredictChar path per char (respecting `in_bracketed_paste`, right-edge `become_tentative` guard, pushing `PendingPrediction`, advancing `predicted_cursor.col`).

Option B (`classify_input_batch` sibling function) was considered but rejected: `classify_input` is the public API tested by anti-regression tests (`classify_bulk_suppressed`, `classify_cjk_3byte_not_bulk`, etc.), so the fix needed to manifest there to allow those tests' expectations to be updated correctly.

### Content Inspection in `classify_input`

After the bracketed-paste block (unchanged, remains first), inside the `if bytes.len() > BULK_SUPPRESS_THRESHOLD` branch:

1. Attempt UTF-8 decode of the entire batch.
2. For each decoded char, check `UnicodeWidthChar::width(ch)`:
   - `Some(1)` → push `(ch, 1)` to batch
   - anything else (`None` = control, `Some(0)` = combining/ZWJ, `Some(2)` = CJK wide) → set `all_single_width = false`, break
3. If `all_single_width && !batch.is_empty()` → return `InputAction::PredictBatch(batch)`.
4. Otherwise → fall through to `return InputAction::BulkSuppressed` (unchanged behavior for non-printable/mixed/escape batches).

The flat `> BULK_SUPPRESS_THRESHOLD` guard remains as the outer condition, so single-char paths (≤ 4 bytes) are completely unaffected.
