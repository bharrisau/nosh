#### D-01 — BUG-E: epoch-start clamp floor (lines 330–346 fix site)

The clamp floor is added as a new field on `PredictionOverlay`, set from
`sync_cursor_from_confirmed` on every fresh epoch start, then used in
`PredictBackspace` and `PredictCursorLeft`.

**Existing clamp pattern** (`predictor.rs` lines 330–347):
```rust
InputAction::PredictBackspace => {
    if self.predicted_cursor.col > 0 {          // ← clamp today: only col 0
        let vacated_col = self.predicted_cursor.col - 1;
        let row = self.predicted_cursor.row;
        self.pending.retain(|p| !(p.row == row && p.col == vacated_col));
        self.predicted_cursor.col = vacated_col;
        self.cursor_motion_pending = true;
    }
}
InputAction::PredictCursorLeft => {
    if self.predicted_cursor.col > 0 {          // ← clamp today: only col 0
        self.predicted_cursor.col -= 1;
    }
    self.cursor_motion_pending = true;
}
```

**Fix shape:** Replace `> 0` with `> self.epoch_start_col` (new field).
Set `epoch_start_col` from `sync_cursor_from_confirmed` when `pending` is empty
and `cursor_motion_pending` is false (the same guard already on lines 244–246).

**CR-01 guard that must NOT regress** (`predictor.rs` lines 243–246):
```rust
pub fn sync_cursor_from_confirmed(&mut self, confirmed: CursorPos) {
    if self.pending.is_empty() && !self.cursor_motion_pending {
        self.predicted_cursor = confirmed;
    }
}
```

**Test analog to copy** — `cr02_backspace_removes_char_prediction_from_overlay`
(lines 1548–1579) and `bug_d_arrow_motion_moves_caret_and_survives_sync`
(lines 1675–1741). The new test must:
1. Confirm 'a' at col N so `sync_cursor_from_confirmed` seeds `epoch_start_col = N`.
2. Press backspace N+1 times — cursor must stop at col N, not col 0.
3. Assert `predicted_cursor.col == N` (clamp floor held).

#### D-02 — Typematic / fast-typing: bulk reclassification (lines 616–628 fix site)

**Current flat rule** (`predictor.rs` lines 625–629):
```rust
if bytes.len() > BULK_SUPPRESS_THRESHOLD {
    return InputAction::BulkSuppressed;
}
```

**Fix shape:** Before the flat guard, inspect the batch: if every byte is
printable ASCII (0x20–0x7E) or valid UTF-8 with `UnicodeWidthChar::width` of 1 or
2, yield multiple `PredictChar` actions (or extend `InputAction` to carry a
`Vec<(char, u16)>`). If ANY byte is a control character, escape byte (0x1B), or
produces an ambiguous/ZWJ/combining width → fall through to `BulkSuppressed`.

**Invariant: bracketed-paste markers stay first** (lines 619–623):
```rust
match bytes {
    b"\x1b[200~" => return InputAction::BracketedPasteStart,
    b"\x1b[201~" => return InputAction::BracketedPasteEnd,
    _ => {}
}
```
The new printable-batch check must come AFTER this block and BEFORE the existing
flat `> BULK_SUPPRESS_THRESHOLD` guard.

**Classify-printable helper to reuse** (lines 678–696):
```rust
pub fn classify_printable(bytes: &[u8]) -> InputAction {
    if let Ok(s) = std::str::from_utf8(bytes) {
        let mut chars = s.chars();
        if let (Some(ch), None) = (chars.next(), chars.next()) {
            match UnicodeWidthChar::width(ch) {
                Some(1) => return InputAction::PredictChar { ch, col_width: 1 },
                Some(2) => return InputAction::PredictChar { ch, col_width: 2 },
                _ => {}
            }
        }
    }
    InputAction::EpochReset
}
```

**Tests that must keep passing** (anti-regression):
- `classify_cjk_3byte_not_bulk` (lines 752–767): '你' = 3 bytes, must not be
  `BulkSuppressed`.
- `classify_bulk_suppressed` (lines 967–974): `b"hello"` = 5 bytes, non-escape →
  current expectation is `BulkSuppressed`. This test will CHANGE under D-02: the
  correct new expectation is five consecutive `PredictChar` actions (or whatever
  the new bulk-printable variant is called).
- `classify_bracketed_paste_start` / `_end` (lines 896–912): must still be
  recognised before the new check fires.

**New tests required by D-06 (fail-before / pass-after):**
- Input `b"aaaaa"` (5 printable bytes) → must yield five predictions in sequence,
  NOT `BulkSuppressed`.
- Input with a mix of printable and `\x1b` → must yield `BulkSuppressed`
  (unchanged).
- `on_input` with a 5-char batch → `predicted_cursor.col == 5` (cursor advanced
  for each char).

#### D-05 — BUG-F: noecho / EpochReset cursor sync (lines 308–374, 576–604 fix site)

**The `reset()` path** (`predictor.rs` lines 493–500):
```rust
pub fn reset(&mut self) {
    self.pending.clear();
    self.become_tentative();
    self.cursor_motion_pending = false;
}
```

**The `EpochReset` arm** (`predictor.rs` lines 369–374):
```rust
InputAction::EpochReset | InputAction::BulkSuppressed => {
    self.reset();
}
```

**Fix shape:** After `self.reset()` in the `EpochReset` arm (or in `reset()`
itself with a caller-supplied confirmed cursor), forcibly sync
`self.predicted_cursor` from the caller's confirmed cursor. The cleanest approach
that avoids disturbing `BulkSuppressed` (which should NOT sync — no confirmed
cursor available at that point) is to add a separate `reset_with_cursor(cursor)`
method and call it only from the `EpochReset` arm in `on_input`, passing
`screen.confirmed_cursor()`. The existing `sync_cursor_from_confirmed` guard
(pending-empty AND no motion) is too conservative for this case — on EpochReset
pending IS being cleared, so a direct assignment is needed.

**Test analog** — `bug_d_noecho_after_reset_is_hidden` (lines 1641–1664) — Enter
resets then typing is hidden. The new BUG-F test must additionally assert that
after the reset, `self.predicted_cursor` equals the confirmed cursor, and that a
subsequent Enter typed in the correct position advances the line as expected.

### `crates/nosh-client/src/screen.rs`

**Decisions:** D-03 (BUG-H: clear-on-connect + blank-cell painting), D-06 (tests)

#### D-03a — Blank-cell painting in `emit_diff`

**Current diff loop** (`screen.rs` lines 445–476 — the core of `emit_diff`):
```rust
for (r, (des_row, phys_row)) in desired.iter().zip(self.physical.iter()).enumerate().take(rows) {
    let row = r as u16;
    for (c, (want, have)) in des_row.iter().zip(phys_row.iter()).enumerate().take(cols) {
        let col = c as u16;
        if want == have {
            continue;  // ← BUG-H: blank Cell == blank Cell, skipped, never clears
        }
        // ... MoveTo + SGR + write char
        out.write_all(s.as_bytes())?;
    }
}
```

The default blank `Cell` has `ch = ' '`, `style = NONE`, `fg = None`, `bg = None`.
When the server sends an ED2 (erase entire display), `compute_diff_runs` in
`server.rs` includes the newly-blanked cells in the diff (they differ from the
baseline). The client applies them to `confirmed` but they remain as
`Cell { ch: ' ', ... }` — identical to the `physical` grid's defaults. The
`want == have` check skips them, so no space character is ever written, and the
prior terminal content bleeds through.

**Fix shape in `emit_diff`:** Change the skip condition: emit a cell if `want !=
have`, OR if `want` is the default cell but `have` is NOT the default cell (i.e.
the cell was previously non-blank and must now be cleared). The simplest
expression: `if want == have { continue; }` is replaced by checking that both the
content AND the physical record agree; a blank `want` that differs only because
`have` has a non-space char must still emit a space. In practice this means
removing the early-continue for cells where `want.ch == ' '` but `have.ch != ' '`.

**Test analog** — `reset_physical_forces_full_repaint` (lines 975–1001) and
`render_after_apply_emits_nonempty_ansi_with_chars` (lines 893–910). The new
BUG-H blank-cell test must:
1. Apply content ('X') to confirmed at (0,0); render → physical shows 'X'.
2. Apply a diff that resets (0,0) back to `' '` (space, like an ED2).
3. Call `render_to_stdout` into a buffer.
4. Assert the buffer contains a space write at (0,0) — NOT an empty buffer
   (the current broken behaviour).

**Adversarial test shape:**
```rust
#[test]
fn bug_h_blank_cell_after_clear_is_emitted() {
    // FAIL BEFORE FIX: space cell matches default physical → skipped → old char bleeds through.
    // PASS AFTER FIX:  space is emitted to overwrite the physical 'X'.
    let mut screen = ClientScreen::new(80, 24);
    // Write 'X' to confirmed and render (physical now has 'X' at (0,0)).
    screen.apply(&make_diff(1, "X"));
    let mut buf1 = Vec::<u8>::new();
    screen.render_to_stdout(&mut buf1).unwrap();
    assert!(String::from_utf8_lossy(&buf1).contains('X'));

    // Now apply a diff that blanks (0,0) back to space (ED-style clear).
    let blank_diff = StateDiff {
        epoch: 2, cols: 80, rows: 24,
        cursor: CursorPos { row: 0, col: 0 },
        runs: vec![DiffRun {
            row: 0, start_col: 0,
            chars: " ".to_string(),
            style: CellStyle(CellStyle::NONE), fg: None, bg: None,
        }],
    };
    screen.apply(&blank_diff);
    let mut buf2 = Vec::<u8>::new();
    screen.render_to_stdout(&mut buf2).unwrap();

    // The output MUST contain a write at (0,0) — a space must be emitted to clear 'X'.
    // The MoveTo(0,0) sequence is ESC [ 1 ; 1 H.
    let output = String::from_utf8_lossy(&buf2);
    assert!(
        output.contains("\x1b[1;1H") || buf2.contains(&b' '),
        "BUG-H: blanking a previously non-blank cell must emit a space (not skip it)"
    );
}
```

#### D-03b — Clear-on-connect (physical reset to emit initial clear)

**Existing `reset_physical` pattern** (`screen.rs` lines 499–502):
```rust
pub fn reset_physical(&mut self) {
    self.physical = Self::make_grid(self.cols, self.rows);
    self.physical_cursor = CursorPos { row: 0, col: 0 };
}
```

**New method shape:** Add `pub fn emit_connect_clear<W: Write>(&mut self, out: &mut W)` that
writes `\x1b[2J\x1b[H` (ED2 + cursor-home) directly to `out`, then calls
`reset_physical()` so the subsequent `render_to_stdout` forces a full repaint from
a known-clean terminal state. The ED2+home is written via `out.write_all`, NOT
via `emit_diff` (the confirmed grid is empty at connect time — there is nothing to
diff against). This is the one narrow exception to the "all output through
render_to_stdout" invariant: it fires once, before the first datagram arrives, and
`reset_physical()` ensures the invariant holds for all subsequent renders.

**Invariant reference from CLAUDE.md / CONTEXT.md:**
> all output to the local terminal goes through `ClientScreen.render_to_stdout()`

The connect-clear is explicitly approved by D-03: "emit a physical clear to the
terminal on connect". Calling it a `pre_render_clear` helper on `ClientScreen`
keeps the intent explicit; `run_pump` calls it once after constructing the screen.

**Call site in `run_pump`** (line 790 is the construction point):
```rust
let mut screen = nosh_client::screen::ClientScreen::new(cols, rows);
// D-03: clear terminal on connect so prior content doesn't bleed through.
let mut init_buf: Vec<u8> = Vec::new();
screen.emit_connect_clear(&mut init_buf);
stdout.write_all(&init_buf).await?;
stdout.flush().await?;
```

### `crates/nosh-client/src/main.rs`

**Decision:** D-04 (per-keystroke RTT rework of D-17-02a latency hook)

#### D-04 — Per-keystroke RTT (extend existing HashMap hook)

**Current hook — enqueue side** (`main.rs` lines 1143–1153):
```rust
// D-17-02a latency instrumentation: record enqueue time for the
// current prediction epoch so the datagram arm can measure confirmation
// latency when this epoch is confirmed.
let epoch_required = predictor.prediction_epoch();
predict_enqueue_times.entry(epoch_required).or_insert_with(Instant::now);
tracing::debug!(
    target: "nosh::predict",
    event = "predict",
    epoch_required,
    "keystroke predicted"
);
```

**Current hook — confirm side** (`main.rs` lines 991–1011):
```rust
let epoch_before_cull = predictor.confirmed_epoch();
predictor.cull(&screen, diff.epoch, rtt_ms);
// ...
let epoch_after_cull = predictor.confirmed_epoch();
if epoch_after_cull > epoch_before_cull {
    if let Some(enqueued_at) = predict_enqueue_times.remove(&epoch_after_cull) {
        let latency_ms = enqueued_at.elapsed().as_millis() as u64;
        tracing::debug!(
            target: "nosh::predict",
            event = "confirm",
            epoch = epoch_after_cull,
            latency_ms,
            "prediction confirmed"
        );
    }
    // Prune stale enqueue entries (epochs that were reset/culled).
    predict_enqueue_times.retain(|&k, _| k > epoch_after_cull);
}
```

**Fix shape:** The current hook records one timestamp per `prediction_epoch` (a
group-level epoch). D-04 reworks to per-keystroke RTT: record the
`Instant::now()` for EACH forwarded keystroke batch, keyed by a monotonic
keystroke counter or by `(epoch, pending_len_snapshot)`. On the confirm side,
match the confirmed batch to its timestamp. The HashMap type stays
`HashMap<K, Instant>`; only the key type and insertion logic change. The
`tracing::debug!` log format mirrors the existing `"prediction confirmed"` record
and adds a `keystroke_id` field. The `predict_enqueue_times.retain` pruning
pattern is preserved to prevent unbounded growth.

**Existing map declaration to extend** (`main.rs` lines 826–827):
```rust
let mut predict_enqueue_times: std::collections::HashMap<u64, Instant> =
    std::collections::HashMap::new();
```

Change the key from `u64` (epoch) to a per-keystroke monotonic counter or a
`(u64, u64)` pair; keep the same variable name and initialization location.

#### ED/clear handling audit (read-only for planner)

The server's ED2 implementation (`terminal.rs` lines 606–612):
```rust
2 => {
    // Erase entire display
    for row in &mut self.grid {
        for cell in row.iter_mut() {
            *cell = Cell::default();    // ch = ' '
        }
    }
}
```

The `compute_diff_runs` function in `server.rs` (lines 212–264) diffs changed cells
against the last-acked snapshot. After ED2, every cell is `Cell::default()`;
cells that were already default in the snapshot are equal → skipped; cells that
were non-default are included in the diff with `ch = ' '`. So the server DOES emit
blanked cells in the diff. The bug is on the client `emit_diff` side (see D-03a
above). **No server edit is required unless a reproduction test contradicts this.**

If a reproduction test is written and it reveals the server incorrectly omits
blank-space runs for rows that contained content, the fix site in server.rs would
be `compute_diff_runs` (lines 229–233): the `base.map(|b| b == cell).unwrap_or(false)`
skip condition may be comparing a space cell to a baseline space cell and eliding
the run. The test pattern to copy for a server-side fix is `erase_in_display_2_clears_all`
(`terminal.rs` lines 1039–1049).

### Adversarial test structure (D-06 — applies to all new tests)

Every new test must embed the fail/pass contract in its doc-comment and assertion
messages. Copy this verbatim pattern from `cr02_backspace_removes_char_prediction_from_overlay`
(predictor.rs lines 1542–1545) and `bug_d_printable_visible_immediately_on_keystroke`
(lines 1581–1584):

```rust
/// BUG-X regression: <symptom sentence>.
///
/// Before the fix: <what the broken code did>.
/// After the fix:  <what the correct code does>.
#[test]
fn bug_x_description() {
    // FAIL BEFORE FIX: <what assert fails>.
    // PASS AFTER FIX:  <what assert passes>.
    // ...
    assert!(
        <condition>,
        "BUG-X: <human-readable failure description>"
    );
}
```

### Screen invariant (applies to all D-03 work)

All display output goes through `ClientScreen::render_with_predictor` (or its
wrappers). The connect-clear (`emit_connect_clear`) is the only permitted
exception and is written as a method on `ClientScreen` that also calls
`reset_physical()`, preserving the invariant for all subsequent renders.

**Source:** `screen.rs` lines 349–351 (the existing single-path comment) and
CONTEXT.md § Specific Ideas.

### CR-01 guard (applies to D-01 and D-05)

`sync_cursor_from_confirmed` (predictor.rs lines 243–246) must only update
`predicted_cursor` when `pending.is_empty() && !cursor_motion_pending`. D-01's
epoch-start column recording must use the SAME guard — only capture `epoch_start_col`
in the same condition block, not unconditionally. D-05's forced sync after
`EpochReset` BYPASSES this guard (pending is being cleared), so it uses a direct
assignment inside `reset_with_cursor()`.

### Render path for all display (applies to D-03b call site in main.rs)

Buffer into `Vec<u8>`, then `stdout.write_all(&buf).await` + `stdout.flush().await`,
with `screen.reset_physical()` on error. See `run_pump` lines 1014–1025 for the
established pattern.
