<objective>
Fix two predictor cursor-tracking defects from Phase 17 validation, both pure shared client logic in `predictor.rs`:

- **D-01 (BUG-E):** `PredictBackspace` and `PredictCursorLeft` clamp at col 0, letting the predicted caret walk past the prompt start. Record the confirmed cursor column at the start of each fresh input epoch as a clamp floor (`epoch_start_col`) and clamp there instead.
- **D-05 (BUG-F):** when a noecho epoch ends (`EpochReset`), the predicted caret can be stale, so the Enter after a `read -s` prompt does not advance the line. Sync the predicted caret from the confirmed StateDiff cursor on `EpochReset`.

Purpose: these are real daily-driver correctness bugs that make prediction worse than no prediction in the affected cases (D-15-01 "never worse than no prediction" contract).
Output: modified `predictor.rs` with both fixes and two fail-before/pass-after adversarial tests (D-06).
</objective>
