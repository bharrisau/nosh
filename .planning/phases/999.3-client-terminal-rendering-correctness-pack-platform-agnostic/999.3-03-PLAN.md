<objective>
Fix BUG-H (D-03), the clear-on-connect / blank-cell defect, entirely on the client in `screen.rs` (PATTERNS.md confirmed the server already emits correct space diffs — do NOT touch the server in this plan):

- **D-03a:** `emit_diff`'s `if want == have { continue; }` skips cells that are blank in both grids. After an ED2/Ctrl-L clear, the server diff blanks cells to spaces; the client applies them to `confirmed` but they equal the `physical` default, so no space is written and the prior char bleeds through. Change the emission so a cell that is blank in `want` but non-blank in `have` (`have.ch != ' '`) still emits a space.
- **D-03b:** add `emit_connect_clear` to write `\x1b[2J\x1b[H` (ED2 + cursor-home) then `reset_physical()`, so the terminal is cleared on connect and the first render repaints from a clean state. This is the single sanctioned exception to the render-only invariant; it is a ClientScreen method that itself resets physical, preserving the invariant downstream.

Purpose: prior terminal content no longer bleeds through on connect and Ctrl-L actually clears the screen.
Output: modified `screen.rs` with both fixes and adversarial tests (D-06).
</objective>
