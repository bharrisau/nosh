# Phase 999.3 Plan 03: BUG-H blank-cell emission + connect-clear Summary

Fixes BUG-H (clear-on-connect / Ctrl-L bleed-through) entirely on the client in `screen.rs`. Two deliverables: an adversarial test verifying D-03a (blank-cell emission already correct), and a new `emit_connect_clear` method implementing D-03b (physical clear on connect).

## What Was Built

**D-03a verification (Task 1):** The existing `emit_diff` correctly emits a space when `want.ch == ' '` and `have.ch != ' '` (they are unequal, so the early-continue is NOT taken). The plan explicitly anticipated this: "If the existing equality already handles it, the test will pass without code change." The adversarial test `bug_h_blank_cell_after_clear_is_emitted` documents and verifies this invariant with a BUG-H tag and Before/After doc-comment. No code change to `emit_diff` was needed.

**D-03b implementation (Task 2):** Added `pub fn emit_connect_clear<W: Write>(&mut self, out: &mut W) -> std::io::Result<()>` to `ClientScreen`. The method writes `\x1b[2J\x1b[H` (ED2 + cursor-home) then calls `reset_physical()`, so the terminal is cleared and the physical model is re-aligned before the first datagram renders. This is the single sanctioned exception to the render-only invariant (CONTEXT.md), documented in the method's doc-comment referencing D-03.

The adversarial test `bug_h_connect_clear_emits_ed2_home_and_resets_physical` verifies:
1. The clear buffer contains `\x1b[2J` and `\x1b[H`
2. `reset_physical()` was called (next render is a full repaint)

**Server:** NOT modified. `PATTERNS.md` and `phase17-round2-triage.md` confirmed the server's `compute_diff_runs` correctly includes blank cells in ED2 diffs. No D-03 contingency triggered.

### Analysis deviations

**D-03a — no emit_diff change required (expected per plan)**

During TDD RED, the test for D-03a passed immediately without any code change. Root-cause analysis confirmed:
- `want.ch == ' '` and `have.ch == 'X'` → `want != have` (Cell derives PartialEq on all fields) → the cell IS emitted correctly.
- The plan explicitly anticipated this: "If the existing equality already handles it, the test will pass without code change."
- The test still has meaningful value as regression documentation.

The plan's description of the bug ("they equal the `physical` default") refers to the scenario where `want == have == Cell::default()` (both blank). In this case, nothing is emitted — but from nosh's perspective, the physical model says the terminal is blank too. The mismatch is between nosh's physical model and the actual physical terminal (which has old content from before nosh started). This is correctly fixed by D-03b (emit_connect_clear) rather than D-03a.
