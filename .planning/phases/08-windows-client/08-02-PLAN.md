<objective>
Make the nosh client cross-compile to Windows and behave correctly there. Bump crossterm
0.28→0.29 (events + event-stream, NOT use-dev-tty), add `futures`, and split the platform
behavior in `main.rs`: a `--identity-file` flag wired to `ClientIdentity::from_identity_file`
(opt-in everywhere; the only path on Windows, default `%USERPROFILE%\.ssh\id_ed25519`); the
Unix `SIGWINCH`/`SIGINT` signal plumbing replaced by a `#[cfg]`-split resize trigger
(SIGWINCH on Unix, crossterm `EventStream` `Event::Resize` on Windows, both feeding the
existing `Message::Resize` with ~40 ms coalescing) and the cross-platform `tokio::signal::ctrl_c`
for the reconnect-window quit; and `TERM`/`LANG` defaulting in the forwarded env. Prove it with
`cargo check --target x86_64-pc-windows-gnu` (the D-01 automated gate) and ship a documented,
NON-BLOCKING human Windows test procedure (D-02). Mark the phase `human_needed`.

Purpose: This is the half of WIN-01 that proves the Windows build (the cross-compile gate)
plus WIN-03 (raw mode + resize via console events, not SIGWINCH) and WIN-04 (locale). It
depends on Plan 01's `from_identity_file` and ssh-agent gating already being in place.
</objective>
