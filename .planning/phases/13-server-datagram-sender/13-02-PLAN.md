<objective>
Wire the server datagram sender into the session pump (SYNC-03). Add a `diff_interval` tick arm (~16ms) and an epoch-ack `read_datagram` arm to BOTH `run_session` and `run_reattach_session`, implementing the acked-epoch model (D-13-01): each tick diffs the current terminal grid against the last CLIENT-acked snapshot and sends one coalesced `StateDiff` datagram. Datagrams are suppressed until a `ResumeComplete` gate fires — immediately for fresh `run_session`, after replay completes for `run_reattach_session` (D-13-03). The reliable `PtyData` stream is untouched (D-13-04 additive).

Purpose: This closes the server→client datagram channel end-to-end. It is the deliverable of Phase 13; Phase 14 builds the real client renderer on top.

Output: One modified file (`server.rs`) with a shared diff-tick helper and matching select! arms in both session functions. Plan 03's integration test proves the datagrams actually arrive at a client.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → server datagram channel | Client-originated datagrams arrive via `conn.read_datagram()` in the session pump and are decoded by `decode_epoch_ack`. QUIC TLS 1.3 authenticates the peer's transport identity; payload content/length is attacker-controllable. |
| server → client datagram channel | Server emits `StateDiff` datagrams carrying terminal grid content. Same QUIC TLS 1.3 confidentiality/integrity as the reliable stream. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Information Disclosure | StateDiff datagram during cold-reattach replay | mitigate | ResumeComplete gate suppresses ALL datagrams until the replay loop completes (`if !resume_complete { continue; }`), so a fresh datagram never applies to a partial replay and never leaks an inconsistent/post-replay grid mid-replay. Enforced by Task 2's post-replay bool flip + Plan 03 suppression test. |
| T-13-05 | Tampering/Replay | incoming epoch-ack value | mitigate | Baseline only advances when `acked_epoch > last_acked_epoch`; stale/duplicate/out-of-order acks are ignored (never regress the baseline). A forged-but-authenticated higher epoch only causes the server to send FEWER cells — the acked-epoch model is self-correcting (next diff re-sends any still-missing cells). No correctness or disclosure impact. (RESEARCH.md Pitfall 2 / Security Domain.) |
| T-13-06 | Denial of Service | pending_deferred unbounded growth (full-screen vim) | mitigate | Deferred-run queue capped at `MAX_RUNS` (4096); excess truncated each tick (Pitfall 3 / Open Question 4). Diff snapshot is O(rows×cols) per tick on an 80×24-class grid — bounded work. terminal_state lock held only microseconds, never across `.await` (Pitfall 1). |
| T-13-07 | Denial of Service | malformed inbound datagram | mitigate | `decode_epoch_ack` returns `Err` on unknown tag / empty / truncated input (Plan 01 T-13-01/02); the arm ignores the error and continues — a flood of malformed datagrams cannot crash or wedge the pump. |

No high-severity threats. Phase 13 introduces no new auth/authz/crypto (RESEARCH.md §Security Domain); the datagram channel reuses QUIC TLS 1.3 per-packet auth. No package installs — no `T-13-SC` supply-chain gate applies.
</threat_model>
