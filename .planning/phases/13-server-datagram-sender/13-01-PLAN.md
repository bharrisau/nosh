<objective>
Add the two foundational pieces Phase 13's server datagram sender depends on, in the two crates that have no overlap with `server.rs`:

1. The client→server **epoch-ack** wire format in `nosh-proto` (`TAG_CLIENT_EPOCH = 0x02`, `ClientEpoch` struct, `encode_epoch_ack`, `decode_epoch_ack`). This realizes the acked-epoch model (D-13-01/D-13-01a): the client tells the server which `epoch` it has applied.
2. A read-only `with_terminal_state` closure delegate on `SessionSlot` so the server connection task can snapshot the grid under the existing lock discipline without the private `terminal_state` field being exposed (RESEARCH.md Open Question 5).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → server datagram channel | Untrusted client-originated bytes arrive via `conn.read_datagram()` and are handed to `decode_epoch_ack`. QUIC TLS 1.3 authenticates the sender's transport identity, but the application payload bytes are still attacker-controllable in content/length. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | `decode_epoch_ack` tag handling | mitigate | Tag check requires first byte == `TAG_CLIENT_EPOCH` (0x02); any other tag (including `TAG_STATE_DIFF` 0x01) returns `Err` and is never deserialized as a ClientEpoch. Enforced by Task 1 test `decode_epoch_ack_rejects_state_diff_tag`. |
| T-13-02 | Denial of Service | `decode_epoch_ack` on empty/truncated input | mitigate | `split_first()` on empty slice returns `Err` (no panic); truncated body returns `Err` from postcard. ClientEpoch is a fixed single-u64 (≤9 bytes) with no length-prefixed Vec, so there is no allocation-amplification vector (unlike StateDiff's `MAX_RUNS`-guarded runs). Enforced by Task 1 tests `decode_epoch_ack_rejects_empty`/`decode_epoch_ack_rejects_bad_body`. |
| T-13-03 | Spoofing/Replay | epoch-ack value | accept | A replayed/stale epoch-ack is harmless: it only carries a monotonic `u64`. The server-side baseline-advance guard (`acked_epoch > last_acked_epoch`, implemented in Plan 02) prevents any baseline regression. No new crypto in this plan; QUIC TLS 1.3 already authenticates the transport peer. (RESEARCH.md Security Domain.) |

No high-severity threats. No new auth/authz/crypto surface introduced (RESEARCH.md §Security Domain). No package installs in this plan — no supply-chain (`T-13-SC`) gate applies.
