<objective>
Build the interruptible PTY reader foundation: a new `pty_io` module that encapsulates a Unix self-pipe + `nix::poll` reader behind a small trait boundary (D-02), plus the `master_raw_fd()` accessors needed to reach the PTY master fd from the session pump without exposing the private `Session::master` field.

This plan creates the contracts and the reusable mechanism. It does NOT wire them into `server.rs` — that is Plan 02. The output of this plan is a self-contained, compilable module that Plan 02 consumes.

Purpose: Replace the un-interruptible `spawn_blocking` read loop (the root cause of Pitfall 6 — `abort()` on an executing `spawn_blocking` task is a no-op) with a reader that exits cleanly within one polling interval when signalled.
Output: `pty_io.rs` (trait + Unix impl), `Session::master_raw_fd()`, `SessionSlot::master_raw_fd()`, nix `"poll"` feature.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| async session pump → blocking reader thread | The async teardown path signals the blocking thread; the only shared mutable state is the self-pipe and the out_tx channel (no shared fd ownership). |
| reader module → PTY master fd | The reader receives the master fd as a copied `i32` for poll readability only; it must never gain ownership of or close that fd (the slot owns it for the orphan lifetime). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Denial of Service | self-pipe OwnedFd handling in pty_io.rs | mitigate | Each pipe end has exactly one `OwnedFd` owner (read end in the blocking closure, write end in the handle); both drop-close exactly once. No manual `libc::close`, no double-close. Verified by acceptance criteria (no close() on master, single ownership). |
| T-10-02 | Tampering | master fd lifetime | mitigate | The reader receives the master fd as a copied `i32`, never an `OwnedFd`; the module contains no `close()` of the master fd. Closing it would SIGHUP the orphaned shell (Pitfall 7) — breaking session persistence. Enforced by acceptance criteria grep. |
| T-10-03 | Tampering (accidental) | PTY master open-file-description flags | mitigate | Self-pipe path sets no `O_NONBLOCK`/`F_SETFL` on the master fd, so the W2 writer's shared dup is unaffected (Pitfall 1). Verified: no `O_NONBLOCK`/`F_SETFL` in pty_io.rs. |
| T-10-SC | Tampering | nix dependency feature change | accept | No new package; only the `"poll"` feature is enabled on the already-vetted `nix` 0.29 (RESEARCH Package Legitimacy Audit: feature-only addition, approved). No legitimacy checkpoint required. |
